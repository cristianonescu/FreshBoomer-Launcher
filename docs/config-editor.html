Despre aplicatie

FreshBoomer este o aplicatie Android care inlocuieste launcher-ul, telefonul si aplicatia SMS cu o interfata simplificata cu butoane mari, text-to-speech si limba romana — conceputa pentru utilizatorii varstnici. Disponibila in romana si engleza.

Aplicatia preia rolul de launcher implicit (ecranul de start), aplicatie de telefon si aplicatie de SMS pe dispozitivul Android.

Configurarea aplicatiei este stocata intr-un fisier config.json care poate fi importat de pe un URL din setarile aplicatiei.

Cum se acceseaza Setarile din aplicatie

Pe ecranul principal, in partea de stanga jos, exista o mica iconita cu un lacat. Apasa rapid de 5 ori in mai putin de 3 secunde pentru a deschide setarile. Aceasta metoda previne deschiderea accidentala de catre utilizatorul varstnic.

Versiune configurare

Aceste campuri permit aplicatiei sa detecteze daca configurarea de la distanta s-a schimbat si trebuie aplicata.

CampTipDescriere
config_versionIntNumar de versiune, se incrementeaza automat la fiecare modificare in editor. Aplicatia compara acest numar cu cel stocat local si aplica configurarea doar daca versiunea e mai mare.
config_updated_atString (ISO 8601)Data/ora ultimei modificari (ex: 2026-04-06T14:30:00+00:00). Folosit ca tiebreaker cand versiunile sunt egale.

Contacte rapide (quick_contacts)

Contactele rapide sunt afisate ca butoane mari pe ecranul principal. Utilizatorul poate suna pe cineva cu o singura apasare. Daca un contact nu are poza setata, se afiseaza initialele numelui pe un fundal colorat.

Campul quick_contacts este un array de obiecte, fiecare cu:

CampTipDescriere
nameStringNumele contactului afisat pe buton
phoneStringNumarul de telefon (format international sau local)
photo_base64String (optional)Poza contactului codificata in base64 (JPEG). Editorul redimensioneaza automat la max 200x200px. La import, aplicatia decodifica si salveaza local.
photo_mimeString (optional)Tipul MIME al pozei (ex: image/jpeg). Necesar daca photo_base64 este prezent.

Contacte de urgenta (emergency_contacts)

Contactele de urgenta sunt persoanele care vor fi notificate prin SMS daca functia de monitorizare a inactivitatii este activata si utilizatorul nu a interactionat cu telefonul pentru o perioada configurata.

Campul emergency_contacts este un array de obiecte, fiecare cu:

CampTipDescriere
nameStringNumele persoanei de contact
phoneStringNumarul de telefon la care se trimite alerta SMS

Monitorizare inactivitate

Aceasta functie trimite automat un SMS de alerta catre toate contactele de urgenta daca utilizatorul nu a interactionat cu telefonul pentru un numar configurat de ore (implicit: 12 ore). Este dezactivata implicit.

Cerinte:

- Campul inactivity_monitor_enabled trebuie setat pe true

- Cel putin un contact de urgenta trebuie configurat in emergency_contacts

Ce se considera interactiune:

- Orice atingere a ecranului

- Revenirea pe orice ecran al aplicatiei (inclusiv deblocarea telefonului)

- Conectarea incarcatorului

Mesajul SMS trimis: "FreshBoomer: [porecla] nu a folosit telefonul de peste [ore] ore. Verificati daca este in regula."

Alerta se trimite o singura data per perioada de inactivitate — nu se repeta pana cand utilizatorul interactioneaza din nou cu telefonul.

Text-to-Speech (TTS)

Aplicatia poate anunta vocal diverse evenimente (apeluri pierdute, deschiderea ecranelor etc.).

Motoare disponibile:

- PIPER_SANDA — motor offline de sinteza vocala cu voce romaneasca naturala (recomandat). Modelul se descarca automat la prima utilizare.

- PIPER_LILI — voce alternativa romaneasca offline. Modelul se descarca automat la prima utilizare.

- DEVICE_DEFAULT — motorul TTS implicit al dispozitivului Android.

Viteza vorbirii se poate ajusta cu tts_speech_rate: valori mici (0.5) = lent, valori mari (2.0) = rapid. Implicit: 0.85.

Functionalitati (Features)

Fiecare functionalitate poate fi activata sau dezactivata independent:

SetareCe controleaza
feature_quick_contactsButoanele mari de apel rapid pe ecranul principal
feature_dial_padButonul "Formeaza Numar" — tastatura de apel
feature_contactsButonul "Agenda" — lista de contacte din telefon
feature_messagesButonul "Mesaje" — conversatii SMS
feature_galleryButonul "Galerie" — vizualizare poze
feature_whatsappButonul WhatsApp pe ecranul principal (necesita WhatsApp instalat)

Comportament

SetareCe face
auto_max_volumeSeteaza automat toate volumele la maxim. Util pentru a preveni situatiile in care utilizatorul reduce accidental volumul.
inactivity_timeout_msDupa cate milisecunde de inactivitate revine automat pe ecranul principal (de pe orice alt ecran). Implicit: 20000 (20 secunde).
max_missed_call_announcementsDe cate ori se anunta vocal un apel pierdut de la acelasi numar. Implicit: 3.
call_speaker_delay_msIntarziere (ms) inainte de activarea automata a difuzorului la apel. Implicit: 3000 (3 secunde).
inactivity_monitor_enabledActiveaza monitorizarea inactivitatii si alertele SMS. Implicit: dezactivat.
inactivity_monitor_threshold_hoursDupa cate ore de inactivitate se trimite alerta SMS. Implicit: 12 ore.

Tema

Campul theme_mode controleaza aspectul vizual al aplicatiei:

- SYSTEM — urmareste tema sistemului Android (luminos/intunecat)

- LIGHT — tema luminoasa permanent

- DARK — tema intunecata permanent

Limba / Language

Campul app_language controleaza limba interfetei:

- ro — Romana (implicit)

- en — English

Limba poate fi schimbata si din aplicatie: in ecranul de bun venit (la prima instalare) sau din Setari → Limba. Schimbarea se aplica imediat — aplicatia se recreeaza cu noua limba.

Daca limba este schimbata prin configurarea de la distanta (remote config), aplicatia detecteaza automat schimbarea si se recreeaza.

Cum se foloseste editorul

1. Editeaza setarile in tab-ul Editor.

2. Descarca fisierul JSON din tab-ul Import / Export.

3. Incarca fisierul pe un server web (sau foloseste un serviciu ca GitHub Gist, Pastebin, etc.).

4. In aplicatie: Setari → Configurare Avansata → Importa Configurare din URL → lipeste URL-ul.

Alternativ, poti importa un JSON existent in editor folosind tab-ul Import / Export (lipeste textul sau incarca un fisier .json).

Exemplu config.json complet

Un fisier JSON valid pentru FreshBoomer arata astfel:

{
  "config_version": 3,
  "config_updated_at": "2026-04-06T14:30:00+00:00",
  "user_nickname": "Mamaie",
  "theme_mode": "SYSTEM",
  "tts_enabled": true,
  "tts_engine": "PIPER_SANDA",
  "tts_speech_rate": 0.85,
  "feature_quick_contacts": true,
  "feature_dial_pad": true,
  "feature_contacts": true,
  "feature_messages": true,
  "feature_gallery": true,
  "feature_whatsapp": false,
  "app_language": "ro",
  "auto_max_volume": true,
  "inactivity_timeout_ms": 20000,
  "max_missed_call_announcements": 3,
  "call_speaker_delay_ms": 3000,
  "inactivity_monitor_enabled": false,
  "inactivity_monitor_threshold_hours": 12,
  "emergency_contacts": [
    { "name": "Fiul", "phone": "0712345678" }
  ],
  "quick_contacts": [
    { "name": "Maria", "phone": "0723456789", "photo_base64": "...", "photo_mime": "image/jpeg" },
    { "name": "Ion", "phone": "0734567890" }
  ]
}

Toate campurile sunt optionale — valorile lipsa vor folosi valorile implicite.
